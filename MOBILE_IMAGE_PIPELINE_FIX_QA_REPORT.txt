Mobile Mission Image Pipeline Fix QA

Method change:
- Previous attempts tried compositor/layer fixes.
- This patch instead reduces image decode/raster pressure, which matches the LOG002 and lightbox-open flicker pattern.

What changed:
1. Mission Log images receive loading=lazy, decoding=async, and fetchpriority hints.
2. Images near the viewport are pre-decoded using IntersectionObserver before they appear.
3. Lightbox thumbnails are no longer all loaded/decoded immediately on open.
4. Only the current and nearby thumbnails are hydrated first; the rest load after idle time.
5. Touch devices use auto thumbnail scrolling rather than smooth scroll inside the lightbox.
6. Mobile blur, ambient background, solar-system layer, glass cards, and lightbox blur are preserved.

Why this should address the video:
- LOG002 contains the heaviest grid of images.
- The original lightbox built 14 thumbnail images with full-size sources immediately.
- That creates a burst of image decoding/rasterization on mobile.
- This patch changes the image pipeline rather than removing visual effects.
