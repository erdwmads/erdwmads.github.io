Interface 2046 Stability Audit

Problem reported:
- In Edge/Chromium, vertical white/grey bands appeared on Mission Log status cards and also crossed nearby small pills/tags.
- The previous fix reduced backdrop-filter on status cards, but the visible bands remained.

Actual root cause corrected in this pass:
1. Several Space Mode components still used lateral/diagonal highlight gradients.
   These can look like vertical stripes, especially during scrolling.
2. body::after used mix-blend-mode: screen over a fixed interface overlay.
   This is compositor-sensitive in Chromium.
3. Small cards, pills, tags, and secondary buttons still used glass/backdrop-style effects.
   Edge/Chromium can repaint these unevenly over fixed/animated backgrounds.

Corrections:
- Removed lateral/diagonal highlight gradients from affected Space Mode pills, tags, badges, status cards, project-progress panels, mission-record fields, and secondary buttons.
- Disabled backdrop-filter on these small, repeated surfaces.
- Rebuilt them with stable vertical gradients and solid background colours.
- Removed pseudo-element glints from affected surfaces.
- Disabled mix-blend-mode on the fixed HUD overlay in Space Mode.
- Preserved the overall Interface 2046 style, Solar System animation, Mission Log scientific records, and Firefox first-paint theme guard.

Files included:
- All HTML pages, to keep the early Firefox theme guard stable.
- assets/css/style.css
- assets/js/theme.js

Remaining recommendation:
- If another Edge-only rendering artefact appears, the next step should be to create a dedicated no-backdrop compositor-safe Space Mode profile for the entire site. This patch already applies that approach to the high-risk repeated elements.
