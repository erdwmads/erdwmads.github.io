# Writing Style

The site content should be written in English. Conversations with the site owner can be in Chinese or Japanese, but public website prose should remain English unless explicitly requested otherwise.

## Mission Log Voice

Mission Log prose is a technical research archive, not a casual blog and not a formal paper.

Use:

- first person when describing performed work
- concrete dates, places, materials, instruments, and collaborators
- process-first titles
- restrained technical language
- short paragraphs

Avoid:

- promotional language
- emotional overstatement
- generic academic homepage filler
- rewriting the site into a generic portfolio

## Paragraph Pattern

A normal entry should use two or three paragraphs:

1. What was done, where, and with what material/instrument.
2. What was observed, completed, compared, or found problematic.
3. What will happen next, including constraints or schedule if relevant.

## Title Pattern

Use process-based titles:

```text
Mission Log 007 - Diamond wire saw cutting and sample preparation
```

Good title vocabulary:

- resin embedding
- demoulding
- dry polishing
- ion milling preparation
- SEM observation
- EPMA mapping
- diamond wire saw cutting

Avoid conclusion-heavy titles unless the entry actually reports a scientific conclusion.

## Status Text Rules

- `latestNote`: one completed-fact sentence.
- `stage`: short current process state.
- `stageNote`: concrete explanation of the stage.
- `questionShort`: stable project label.
- `questionNote`: current scientific or technical question.
- `nextStep`: short action phrase.
- `nextNote`: one sentence with schedule, collaborator, equipment, or constraint.

## Captions And Alt Text

Captions use per-entry numbering:

```text
Fig. 1 - Resin-embedded CI chondrite samples in the vacuum chamber.
```

Caption can include archive-specific details such as location, left/right sample identity, instrument, or preparation history.

Alt text should be visual and direct. Do not simply duplicate the caption when a shorter visual description is better.

## Tags

Use roughly five to seven tags per entry.

Recommended mix:

- material
- operation
- instrument
- problem or observation
- next-step context

Tags should be short noun phrases, not sentences.
