# Website Maintenance Templates

These templates are for sending material to Codex. The user does not need to edit website files manually.

## Add A Mission Log Entry

Send whatever material is available. Codex will decide the final structure, log number, fields, figure layout, alt text, captions, and insertion into the private Mission Log archive.

Useful material:

```text
Date:
What happened:
Why it matters:
Current stage:
Current question:
Next step:
People / lab / instrument:
Images:
Image captions or rough notes:
Any wording that must be preserved:
```

Codex must:

- Convert the material into one coherent Mission Log entry, or split it into multiple entries if the process clearly contains separate phases.
- Update `C:\Users\tsuku\Desktop\Grad Research\Orgueil Grad Data\Website Archive\mission-log.private.json`.
- Keep the page rendered through `src/components/MissionLogShell.astro` and the lazy Mission Log scripts.
- Keep latest-first Mission Log order and chronological Mission Index order.
- Back up every new original image to `C:\Users\tsuku\Desktop\Grad Research\Orgueil Grad Data\事前準備`.
- Verify image filenames exist under `public/assets/img/mission-log/`.
- Use `assets/img/mission-log/...` paths for Mission Log images.
- Generate the encrypted archive with the passphrase held only in process memory or a one-time local handoff.
- Run the protected tests, build, PPT export, and site check before committing website changes.
- Never commit the private JSON or passphrase.

## Add Papers

Send bibliographic material in any form: DOI, title, copied citation, summary, or notes. Codex will normalize it into `src/data/papers.ts`.

Useful material:

```text
Title:
Authors/year:
Why it is relevant:
Topic/category:
Tags:
Paper Shelf filters, if known:
Priority or reading status:
```

Codex must:

- Add or update entries in `src/data/papers.ts`.
- Keep tags short enough for mobile.
- Add `filters` for every paper using existing Paper Shelf filter IDs: `dolomite`, `ci-orgueil`, `ryugu-bennu`, `methods`, `chronology`.
- Keep `tags` as reader-facing labels and `filters` as button-matching categories.
- Run `npm run check`.

## Replace Or Add Images

Send the image files and describe their role. Codex will place/rename/reference them.

Useful material:

```text
Image role:
Which page or entry:
Preferred caption:
Important visual details:
Can the file be resized/compressed for web display? yes/no
```

Codex must:

- Keep original research/photography images unless destructive replacement is explicitly approved.
- Add thumbnails or CDN display URLs when useful.
- Keep Mission Log images under `public/assets/img/mission-log/`.
- Keep `loading="lazy"` and `decoding="async"` for Mission Log images.
- Run `npm run audit:images` after larger image changes.
- Run `npm run check`.

## Create A Presentation From Website Content

Send the target audience and presentation goal. Codex can use `dist/ppt-data.json`, generated by `npm run export:ppt-data`, for public paper/project data and the private Mission Log JSON for protected research records.

Useful material:

```text
Audience:
Duration:
Language:
Main claim:
Required figures:
Required papers:
Strict slide count:
```

Codex must:

- Build the deck from the private Mission Log and public Paper Shelf data without copying protected entries into `dist/ppt-data.json`.
- Keep research claims aligned with the website source data.
- Ask for missing scientific details only when needed.
