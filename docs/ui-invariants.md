# UI Invariants

This site is not a generic academic homepage. Its identity is a cosmic mineralogy research archive with an Interface 2046 / Mission Log language.

## Preserve These Concepts

- Mission Log
- Mission Index
- SOL numbering
- Current Focus
- Analytical Pathway
- Interface 2046
- Cosmic Mineralogy
- Orgueil CI1 chondrite
- Dolomite in Orgueil CI1

Do not remove these concepts during refactors unless the owner explicitly changes the site direction.

Mission Log content is private. The public site may preserve its shell and visual language, but detailed entries are available only from the encrypted archive after the protected unlock flow.

Maintain entries in `C:\Users\tsuku\Desktop\Grad Research\Orgueil Grad Data\Website Archive\mission-log.private.json`, publish only the encrypted archive, and never restore a plaintext source route under `src`, `public`, or `dist`.

## Theme Behavior

The initial theme is applied before the CSS finishes loading. Keep that early theme preload behavior to avoid flashing.

Current assumptions:

- `localStorage` key: `mads-theme`
- theme values: `light`, `space`
- attributes/classes used by CSS: `data-theme`, `data-theme-space`, `body.space-mode`

Do not move initial theme selection to the end of the page.

## Ambient FX

Ambient effects should be opt-in and low power by default.

Current assumptions:

- `localStorage` key: `madsAmbientFxEnabled`
- default is off unless the user enables FX
- `prefers-reduced-motion: reduce` disables FX
- FX button is generated by `ambient-space.js`, not hard-coded in HTML
- OFF should remove the ambient layer, not merely hide it

Keep the gold ratios unless intentionally redesigning ambient visuals:

- stardust gold ratio: 18%
- meteor gold ratio: 7%

## Desktop And Mobile Mission Log Images

Desktop:

- lightbox behavior is allowed
- thumbnail hydration is defensive and should not be casually simplified

Mobile/coarse pointer:

- Mission Log images are figures, not buttons
- lightbox is disabled
- captions are visible inline
- hover/open affordances should be suppressed

## Floating UI Placement

Avoid viewport-height floating placement for persistent controls. Use stable bottom/right offsets and maintain clearance from existing Target / Phase / Mode cards.

## Light Mode And Space Mode

Light Mode is not just an inverted Space Mode. Some controls need a pale glass / champagne skin in Light Mode and a darker capsule in Space Mode.

When changing a component, check both themes explicitly.

## Stylesheet Ownership

- `style.css`: legacy compatibility rules only; do not append new component work here.
- `tokens.css`: current Light/Space theme tokens.
- `shell-evolution.css`: header, navigation, theme controls, mobile shell, and supporting HUD visibility.
- `research-log-evolution.css`: Research Log dashboard, protected gate, and graduation research surfaces.

The evolved styles remain globally loaded because soft navigation replaces the page body without replacing the document head. A future page-specific split must first teach soft navigation to synchronise stylesheet links.
